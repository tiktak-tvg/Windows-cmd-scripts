Какие команды вы используете для диагностики сети?
ipconfig    - используется для отображения текущих настроек протокола TCP/IP и для обновления некоторых параметров, задаваемых при автоматическом 							                   конфигурировании сетевых интерфейсов при использовании протокола DHCP
ping           -  для обмена служебной и диагностической информацией в сети используется специальный протокол управляющих сообщений ICMP, если при обмене сообщениями возникает какая-либо проблема, то утилита  выведет информацию для ее диагностики.
arp             - используется для получения аппаратных адресов сетевых адаптеров (MAC-адресов) как на локальном, так и на удаленном компьютере.
route print - используется для просмотра и модификации таблицы маршрутов на локальном компьютере.
netstat       -  используется для отображения TCP и UDP -соединений, слушаемых портов, таблицы маршрутизации, статистических данных для различных протоколов. 
nslookup  -  является классическим средством диагностики сетевых проблем, связанных с разрешением доменных имен в IP-адреса. 
tracert       -  классическая утилита трассировки маршрута до заданного узла, по-прежнему остается наиболее часто используемым инструментом сетевой диагностики. 									.
